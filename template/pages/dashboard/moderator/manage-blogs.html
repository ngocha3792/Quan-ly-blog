Tiêu đề bài viết

Tác giả:

Đây là nội dung mô phỏng của bài viết để Moderator có thể đọc và đánh giá trước khi duyệt. Trong thực tế, nội dung này sẽ được load từ database dựa vào ID của bài viết.

Dưới đây là một ví dụ về nội dung bài viết dài hơn để kiểm tra khả năng cuộn trang (scroll) của Modal. Khi một bài viết có nhiều chi tiết, hình ảnh hoặc các đoạn mã nguồn, Moderator cần phải kiểm tra kỹ lưỡng trước khi quyết định duyệt (Approve) hoặc từ chối (Reject).

1. Tại sao tính năng này quan trọng?

Việc kiểm duyệt bài viết giúp đảm bảo nền tảng DevBlog luôn giữ được chất lượng nội dung tốt nhất. Các bài viết rác (spam) hoặc copy từ nguồn khác mà không ghi nguồn sẽ bị loại bỏ.

2. Yêu cầu định dạng bài viết

  • Tiêu đề phải rõ ràng, không giật tít câu view (clickbait).
  • Nội dung không chứa ngôn từ kích động, thù địch hoặc spam.
  • Các đoạn code cần được đặt trong thẻ Code block (pre/code) một cách hợp lệ.

3. Ví dụ đoạn code hợp lệ

function isContentValid(content) {
  // Bài viết phải có ít nhất 500 ký tự
  if (content.length < 500) return false;
  
  // Không được chứa các từ ngữ cấm
  if (containsSpam(content)) return false;
  
  return true;
}

console.log("Hello Moderator! Chúc bạn một ngày làm việc vui vẻ.");

Bên cạnh đó, việc bố cục bài viết hợp lý cũng được đánh giá cao. Bài viết nên có các thẻ tiêu đề (Heading) từ H2, H3 để chia nhỏ nội dung, giúp người đọc dễ theo dõi hơn.

Cảm ơn tác giả đã đóng góp nội dung hữu ích cho cộng đồng của chúng ta!
